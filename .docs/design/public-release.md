# Public Release

## Scope
- In: local public staging repo, release artifact staging, GitHub Releases upload flow.
- Out: source-history rewriting, GitHub repository creation, CI policy.

## Vocabulary
- **Source checkout**: workspace root, the working source of truth.
- **Public staging**: `continuity-public\`, a generated copy intended to be its own public Git repo.
- **Release assets**: MSI, portable zip, standalone zip, and `SHA256SUMS.txt` under `continuity-public\releases\<version>\`.

## Invariants
- `continuity-public\` is generated by `scripts\sync-public.ps1`; do not hand-edit generated copied files there when the source exists in the source checkout.
- The source checkout ignores `continuity-public\`; the public staging folder may contain its own `.git\` after `git init`.
- `scripts\sync-public.ps1` preserves `continuity-public\.git\`, `.vscode\`, and root `*.code-workspace` files, then refreshes generated contents.
- Public staging excludes private development docs, generated docs, `trace-guide.md`, tests/benches directories, hooks, perf snapshots, and release build internals.
- `continuity-public\releases\` is ignored by the public repo. Release assets are uploaded to GitHub Releases; they are not committed.
- The release version comes from `crates\app\Cargo.toml`.

## Distribution Modes
- **MSI**: normal install; Start Menu, optional desktop shortcut, uninstall, upgrade, and file association support.
- **Portable zip**: `continuity.exe` plus `data\`; double-clicking the exe auto-detects `data\` and keeps settings, themes, keymap, database, tutorial state, and backups in the extracted folder.
- **Standalone zip**: only `continuity.exe`; settings, themes, keymap, database, and backups use normal Windows AppData locations.

## Workflow
1. Update source, docs, version, changelog, and license in the source checkout.
2. Build artifacts from the source checkout:

```powershell
cargo xtask release --skip-sign
```

3. Refresh public staging:

```powershell
.\scripts\sync-public.ps1
```

4. In `continuity-public\`, commit and push source changes through the public repo.
5. Tag the public repo and upload release assets with GitHub CLI:

```powershell
git tag v<version>
git push origin main
git push origin v<version>

gh release create v<version> `
  releases\<version>\continuity-<version>-setup.msi `
  releases\<version>\continuity-<version>-portable.zip `
  releases\<version>\continuity-<version>-standalone.zip `
  releases\<version>\SHA256SUMS.txt `
  --title "Continuity <version>" `
  --notes "<short release notes>"
```

## Key Files
| File | Responsibility |
|---|---|
| `scripts\sync-public.ps1` | Rebuilds `continuity-public\` and release zips/checksums. |
| `scripts\public\README.md` | Public README template copied to `continuity-public\README.md`. |
| `crates\app\Cargo.toml` | Release version source. |
| `CHANGELOG.md` | Human release summary. |
| `LICENSE` | Public license; currently MIT. |
| `xtask\src\release.rs` | Release/signing automation invoked by `cargo xtask release`. |
